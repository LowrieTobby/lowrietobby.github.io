集合框架 - Set

Set集合分类

Set 集合包括以下几种类型

  • HashSet
  • TreeSet

其中两种集合都是异步的，HashSet是无序的，TreeSet是有序的(自然顺序或者规定的顺序)

Set集合方法

Set集合接口的方法与Collection接口的方法一致，没有特有的方法

HashSet

无序性与唯一性

HashSet 是根据哈希表结构进行存放元素的集合，其特点就是无序性和唯一性。

由输出结果可知，hs集合内添加了两个”and”元素，但最终只打印出一个，而且打印元素是无序的

如何存放元素

由于 JAVA 在 Object 基类中定义了 hashCode() 方法，这就使得所有的对象都会产生自己对应的 哈希值 ，而 HashSet 集合中，存放元素的排序依据就是按照元素对象的哈希值大小由小到大进行排序并存放的，在执行打印输出的时候，就会按照排序好的顺序打印显示，但这个顺序并不是存放元素时的顺序，所以看起来是乱序的

如何确保唯一性

HashSet 集合当中，确保唯一性的机制是根据哈希值和 equals() 方法来实现的，原理如下：

  1. 当一个元素被存添加进入 HashSet 集合里时，首先会调用元素对象自身的 hashCode() 方法，得到元素的哈希值
  2. 根据该元素的哈希值对集合内所有元素的哈希值进行比较，如果存在相等情况，则不再把该元素添加到集合里；如果不存在相等情况，继续进行后续步骤
  3. 这时接着调用元素对象的 equals() 方法，对集合内所有元素进行比较，看看是否存在相同对象，如果判断结果为存在相同的元素，则不再添加该元素到集合当中，如果判断结果为没有相同的元素，就将该元素存入集合

取出元素

HashSet 集合与其他集合一样，都具有自己的迭代器，使用迭代器的方法加上for循环就能轻松的遍历整个集合

输出的结果任然是按照哈希值排序的

TreeSet

有序性

Comparable 接口

这里介绍 comparable 接口，这个接口的出现就是给对象实现排序规则的，该接口内只有一个 int compareTo(T o) 方法，重写该方法并实现该接口就可以让一个类的对象具有排序规则了

comparable 接口方法列表

返回值类型 方法与方法描述
int compareTo(T o) 将此对象与指定的对象进行比较以进行排序。

由上表可知，compareTo(T o) 方法的返回值类型是 int 类型，返回值的意义如下：

  • 当返回值为 正整数 时，说明比较结果为调用该方法的 this 对象 大于 被比较的 o 对象
  • 当返回值为 负整数 时，说明比较结果为调用该方法的 this 对象 小于 被比较的 o 对象
  • 当返回值为 负整数 时，说明比较结果为俩对象 相同 ，是同一个对象

在实际情况中，我们并不常需要重写该方法，JAVA中的绝大多数类都实现了这个 Comparable 接口，都有对应的比较方法 (一般排序规则都是按照自然顺序排序，如 String 类型的对象是按照ASCII表的顺序进行排序，也就是常说的 [A-Z,a-z] 顺序) 我们只需要调用 compareTo() 方法即可，特殊情况需要自定排序规则时，就重写该方法即可。

有了 Comparable 接口，我们就可以实现该接口并对自己的类的 compareTo(T o) 方法进行重写，就可以实现自定的排序规则

Comparator 比较器
