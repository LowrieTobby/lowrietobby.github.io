JAVA学习 - 集合框架

发表于 2020-08-08

集合框架 Collection

什么是集合

集合是一种按照特定 数据结构 来存放 对象 的一种容器

这种容器本身还是一个对象

集合的共性方法

集合的共有方法即是对集合内元素进行增删改查的基本方法，其中取出集合元素的方法也就是“查”的方法是通过调用集合内部的迭代器来实现的，由迭代器对元素的取出这一动作进行操作

下面列举一些集合的共有方法

方法名称 返回值 方法描述
add(E e) boolean 向集合内添加一个元素
addAll(Collection<E> c) boolean 把目标集合内的元素添加到本集合内
remove(Object o) boolean 从集合中删除指定元素的单个实例
removeAll(Collection<E> c) boolean 删除集合中包含的所有此集合的元素
clear() void 删除集合内所有元素
isEmpty() boolean 判断集合元素个数是否为0
size() int 返回集合元素个数
retainAll(Collection<E> c) boolean 保留此集合中包含在指定集合中的元素
iterator() Iterator<E> 返回包含在该集合中的元素的迭代器
toArray() Object[] 返回一个包含此集合中所有元素的数组

迭代器 Iterator

什么是迭代器

迭代器是用于对集合的元素做取出操作的一个工具

迭代器是一个接口，它规定了取出操作的通用方法

迭代器的出现使得对于不同类型的集合，进行元素取的出操作变得更加简便

以下是迭代器常用的两种方法

方法名称 返回值 方法描述
hasNext() boolean 如果迭代具有更多元素，则返回 true
next() Object 返回迭代中的下一个元素

为什么要用

迭代器的出现使得对集合中的元素操作变得更加的简便，由于集合的种类有很多种，不同的集合类型所用的存储对象的数据结构也不同，若在每一种集合类型中实现一个迭代器，不同的集合类型对于的迭代器有不同的实现方法，则我们就不需要关心每种集合数据的取出是具体如何实现的，我们只需要调用该集合内部定义的迭代器，使用迭代器接口定义的共性方法，就可以更便捷的取出集合中的元素。

简而言之：统一口径，方便取出元素

获取所有元素

如果需要遍历取出集合中所有元素，只需要使用一个for循环即可

常见集合

  • Set
    • HashSet
    • TreeSet
  • List
    • ArrayList
    • LinkedList
    • Vector（过时，与ArrayList相似）

List 与 Set 的区别：

List 集合是有序的，元素可重复（存在索引）

Set 集合是无序的，元素不可重复
